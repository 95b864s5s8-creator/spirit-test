# 用神五行 × MBTI阵营 守护神映射矩阵算法 v2.0

> v2.0 更新：用神计算升级为50条（5日主×5最旺×2身强身弱），身强身弱正式参与用神选择
> v1.1 记录：月老正式入阵，独占木×NF（恋爱脑专属），观音回归水×NF

> 算法核心：用户需要什么能量（用神五行）+ 用户是什么灵魂风格（MBTI阵营）= 匹配最合适的守护神
> 设计原则：算法先行，数量跟随。不是先定神再找位置，而是先算需求再选神。

### 身强身弱如何参与？

v1.0假设用神已内化身强身弱，但实际发现5×5表有多条映射违背八字常规。

v2.0升级为50条映射（5日主×5最旺×2身强身弱）：
- **身强**：能扛，宜"克泄耗"（修剪、泄秀、输出）
- **身弱**：扛不住，宜"生扶"（补充、支撑、通关）

同一个日主+最旺组合，身强身弱可能得出**不同用神**→匹配**不同守护神**。

> 详细推导见 `cure-element-v2.md`

**用神五行 × MBTI阵营 = 5 × 4 = 20种组合**，守护神矩阵不变。

## 三、双轴定位算法

### 核心公式

```
守护神匹配 = f(用神五行, MBTI阵营)
           = LOOKUP(20格矩阵, CureElement, Camp)
           → 返回 主神(PrimaryDeity)
```

### 神明的双属性设计

每位守护神拥有两个轴上的定位：

1. **五行轴（主）**：神明的核心五行属性 → 决定覆盖哪一行的组合
2. **阵营轴（可跨）**：神明的功能调性 → 决定覆盖该行中的哪几列

规则：
- 每位神明必须有一个**主五行**（不能没有，也不能有两个并列的主五行）
- 每位神明可以覆盖**1-2个阵营**（覆盖3个以上会导致个性模糊）
- 每个组合格至少有**1位神明**覆盖（不能有空格）

### 算法伪代码

```javascript
function getGuardianDeity(cureElement, camp) {
    // cureElement: '木'|'火'|'土'|'金'|'水'
    // camp: 'NT'|'NF'|'SJ'|'SP'
    
    // 方案A：直接查表（20条映射，零复杂度）
    return DEITY_MATRIX[cureElement][camp];
    
    // 方案B（进阶）：如果该格有2位候选神，用第15题（核心困境）做tie-breaker
    // const candidates = DEITY_MATRIX[cureElement][camp]; // 可能返回数组
    // return tieBreak(candidates, coreDilemma);
}
```

## 八、用神五行算法（getCureElement）

### 5×5用神映射表

基于"病药原理"的精准反推——**你最旺的那个五行就是你的"病因"，用神就是"药方"**。

| 日主\最旺 | 最旺=木（同类旺） | 最旺=火（我生旺） | 最旺=土（我克旺） | 最旺=金（克我旺） | 最旺=水（生我旺） |
|---------|-----------------|-----------------|-----------------|-----------------|-----------------|
| **日主=木** | 金（劈木成材） | 水（灭火补身） | 木（同伴扛重担） | 水（通关化解） | 土（筑坝扎根） |
| **日主=火** | 水（水火既济） | 木（补火之源） | 火（同类帮扛） | 木（通关化解） | 土（筑坝导流） |
| **日主=土** | 木（疏通培植） | 土（同类帮扛） | 火（火生土暖） | 水（通关化解） | 金（泄土生金） |
| **日主=金** | 火（火炼真金） | 土（泄火补金） | 金（同类帮扛） | 土（通关化解） | 木（疏通掩埋） |
| **日主=水** | 土（筑坝约束） | 水（同类帮扛） | 木（泄水生木） | 金（通关化解） | 火（驱寒暖水） |

### 算法核心逻辑

```
规则1：最旺=克我者（高压摧残）
  身强 → 用神=通关者（身强能扛，化压力为滋养）
  身弱 → 用神=通关者（身弱扛不住，通关化解唯一出路）

规则2：最旺=我生者（过度透支）
  身强 → 用神=我生者的我生者（泄秀变现：才华→成果）
  身弱 → 用神=生我者（回血补充+克制泄耗）

规则3：最旺=我克者（欲望反噬）
  身强 → 用神=通关者（身强担得起，顺畅转化）
  身弱 → 用神=生我者（帮身分担+通关化解）

规则4：最旺=生我者（溺爱窒息）
  身强 → 用神=我生者（泄秀输出，打破舒适区）
  身弱 → 用神=我生者（泄秀输出，化被动吸收为主动输出）

规则5：最旺=同类者（自我固执 / 外强中干）
  身强 → 用神=克我者（修剪收敛）
  身弱 → 用神=生我者（补充能量，给底气）
```

**诊断**：你把"想清楚再做"活成了"想清楚再想清楚"的死循环。你需要的不是更多信息，是一刀砍掉多余选项的智慧。

**调整方案**：

- **长期方向**：建立"3秒决策机制"——小事3秒内决定，大事最多给自己3天。超过3天的事自动降级为"不值得做"
